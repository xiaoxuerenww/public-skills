## Post

因为第一个电面可能是踩着pass/fail的线，所以HR给约了第二轮电面
经典Revenue的题：
refer(amount) -> id
refer(amount, referrer_id) -> id （referrer_amount += amount)
least_k_users_with_min_amount(min_amount) -> list
follow-up问的是怎么返回和referrer有n层关联的的total amount:
get_amount_of_n_associated(referrer_id, n)
比如:
refer(10) -> 0
refer(20, 0) -> 1
refer(30, 1) -> 2
refer(40, 2) -> 3
那每个user的amount是
0 -> 30
1 -> 50
2 -> 70
3 -> 40
所以
get_amount_of_n_associated(0, 3) -> (10 + 20 + 30 + 40)
get_amount_of_n_associated(0, 2) -> (10 + 20 + 30)
以此类推
每个part都讨论了read heavy和write heavy不同的解法。
