# Source catalog for system design material search

Use this catalog to rank sources and broaden searches. The exact availability of pages changes, so always verify live links.

## Interview-shaped sources

- **Hello Interview**: Strong for structured system design and ML system design lessons, delivery frameworks, and problem breakdowns. Good first stop for learning the shape of an answer.
- **Hack2Hire**: Strong for real interview question pages and ML system design examples. Good for matching a prompt to a concrete walkthrough.
- **ByteByteGo**: Strong for general system design patterns, visual explanations, and some machine learning system design chapters. Useful for backend architecture patterns that support ML systems.
- **Exponent**: Useful for mock interviews and product/system design walkthroughs. Quality varies by video and interviewer.
- **Taro**: Useful for interview prep discussion and senior engineer guidance. Usually secondary, not primary evidence.

## YouTube channels and search targets

Prioritize videos that are actual mock interviews, staff engineer walkthroughs, or architecture lessons:

- Hello Interview
- ByteByteGo
- System Design Fight Club
- Exponent
- MLEpath
- Jordan has no life, when relevant to ML interview prep
- Engineering conference talks from company channels

Avoid videos that only promise interview hacks without architecture, metrics, or trade-offs.

## Production engineering blogs

Use these for Staff+ depth, especially when the interview prompt maps to a real platform:

- Meta Engineering: feed ranking, recommendation, integrity, ads, feature systems, model serving.
- Google Research / Google AI / Google Cloud: ranking, retrieval, recommendation, ML infrastructure, LLM systems.
- Netflix Tech Blog: recommendation, personalization, experimentation, ML platforms.
- Uber Engineering: Michelangelo, feature stores, ranking, marketplace ML, real-time ML.
- DoorDash Engineering: ranking, search, recommendations, experimentation, logistics ML.
- Pinterest Engineering: recommendation, retrieval, embeddings, ads, search.
- LinkedIn Engineering: feed, search, ranking, ads, feature infrastructure.
- Spotify Engineering: recommendations, retrieval, candidate generation, experimentation.
- Airbnb Engineering: search ranking, personalization, marketplace ML.
- Discord, Reddit, Cloudflare, Stripe, Databricks, OpenAI, Anthropic: use when the topic matches safety, infra, LLM systems, observability, or data platforms.

## Books, papers, and courses

Use for conceptual depth and reusable frameworks:

- Chip Huyen, `Designing Machine Learning Systems` and the online `Machine Learning Systems Design` materials.
- Stanford CS329S and other ML systems courses.
- RecSys conference papers for recommendation, ranking, candidate generation, and evaluation.
- SIGIR and KDD papers for search, retrieval, ranking, ads, and recommender systems.
- Trust and safety papers for content moderation, active learning, calibration, human review, and policy enforcement.

## Source ranking rubric

Score resources mentally on these axes:

1. **Prompt match**: Does it directly answer this system design question or a close sibling?
2. **Interview utility**: Does it show how to structure a 45 to 60 minute answer?
3. **Technical depth**: Does it cover data, modeling, serving, evaluation, monitoring, and trade-offs?
4. **Credibility**: Is it from a reputable interview platform, company engineering blog, paper, or known practitioner?
5. **Specificity**: Does it include concrete numbers, architecture diagrams, metrics, or failure modes?
6. **Freshness**: Is it recent enough for LLM, retrieval, and modern ML infra topics?

Prefer a slightly less direct source with strong architecture depth over a direct source that is shallow SEO content.
