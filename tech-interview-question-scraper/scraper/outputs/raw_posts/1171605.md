## Post

众所周知，这里的题目表面上看其实都不算特别难，难的从来不是题本身，而是它真正考察的点藏得很深。很多时候你以为自己答到了，但其实未必答到了他们心里的 bar；而这个 bar 到底在哪里，我们到现在也常常没有完全摸清。也正因为这样，才更希望大家都能多留下一点经验。哪怕只是多写一句提醒、多补一个细节，对后来的人来说，可能都会有帮助。祝好运！
1.Q2 prompt 游乐园
千万不要小看任何以前同学给过的 hint，哪怕只是一个很小的点，都可能是至关重要的。
这道题其实完全不是传统的 sys design。是偏product design。题目一上来会直接给你一个 Google Doc，不用code signal。题目和地理之前的一模一样。而且告诉你不要画sys design图，因为图已经给你了，就三部分：client-web server-database。 面试官并没有问end to end system design，上来直接：
define user flow requirement and why
design database schema 和 index
sharing 怎么 work
怎么 scale viral sharing （server，db）， 怎么delete access
long prompt怎么办 ， multi tab怎么办
我自己回头复盘，觉得我 fail 的一个重要原因，是被面试官带着节奏走了。因为一开始对方直接甩给我一个 Google Doc，我当下会有点慌，结果没有坚持把自己原本准备好的 depth 在每一个模块里展开讲出来。所以这里非常想提醒大家：一定要坚持自己的主线。不要因为对方的引导，就把自己的深度丢掉。
还有一个特别重要的点：面试官有时候会跟你说，“哦，你不用打字，我会自己记笔记。”千万不要完全听他的。一定要把你的思路写下来。原因很现实：很多时候面试官自己也会听漏、听错，最后这些误解反而会反映到你的反馈里。所以，一定一定要把关键思路明确写下来，帮自己留痕。
2. Q6 tokenizer。这题和原题其实是一样的，还是那三问。但这道题真的不用写trie。navie longest match （两个loop）就可以。但是，面试官最后可能会问小延展：如何不output连续的-1（合并连着的-1）， 在vocab很小的情况下如何提升你的loop（用 longest dict key length，而不是text length）。 面试时一定要proactive verify你的想法， 解释你的想法，经常性run test，主动写test - 这些都是面试官想看到的。
3.关于 culture。本质上就是在看：你和 Anthropic 的 value 到底有多 align。所以你准备的每一个故事，都一定要主动往他们的 value 上靠。尤其一开始如果问你：Why Anthropic?
第一，要明确说出是哪个 value。不要泛泛地说“我认同你们的文化”，而是要具体说你认同哪一个 value。
第二，一定要解释为什么你觉得这个 value 很重要。这点非常关键。因为他们想看的不是你会不会背 value，而是你是不是真的理解这个 value 背后的原因。如果你还能顺带讲到 Anthropic 过去做过的某些事情，来说明他们确实在践行这个 value，那会更好。
第三，一定要证明你自己一直也在坚持这个 value。也就是说，不只是“我喜欢你们的价值观”，而是要让他们看到：你本身就是这样的人，你过去的行为已经说明了你和公司是契合的
在讲其他的故事时，尤其是“uphold value” 一定要把这件事描述成情境是有压力、有冲突、甚至让你不舒服的，所以你才做出了那个判断、那个行动。
最后，culture 面一定会看你的 critical thinking。他们不只是想听你认同 Anthropic 的 value 或 policy，还会进一步问你：你觉得 Anthropic 还有什么可以提升的地方？不要把回答做成 product feedback。你是不是会去 challenge policy、value、组织实践背后的张力和 tradeoff
