## Post

短評： 70min共4題，整體難度偏中等吧，前兩題接近leetcode easy難度基本上可以速解，
第一題：Given an array of positive integers, calculate how many of its elements have an even number of digits.
思路：此題簡單就不上圖了，基本上每個number計算how many digits就行
第二題：Given a version control system, you can either "switch" to a branch or "push" a file to the current branch. Given an array of logs, return the branch name that contains the largest number of files at the end.
思路：此題也簡單不上圖，基本用一個hash table 存每個branch有多少files就好
第三題：給定一個matrix，matrix裡有obstacle和一個connected的figure（用”F"表示），figure會掉落直到碰到matrix底部或是obstacle，要計算figure最後的位置
以下内容需要积分高于 188 您已经可以浏览
思路：此題似乎在leetcode碰過但忘記哪一題了，解法是找到所有obstacles, 計算figure會掉落的距離，再根據掉落距離update matrix即可

第四題：給一個array of integers，每個數字x代表house要蓋的位置，需要計算an array of intgers of longest segmenet of contiguous house after each house is built.
思路：此題概念類似leetcode 128. Longest Consecutive Sequence，但需要每次加入後計算一次longest sequence，直接使用lc 128解法是O(n^2)會TLE。快一點的解法是用Union-Find概念，house蓋在x的位置的話，確認x-1和x+1位置上有沒有其他house，若都有的話代表左右原本都有segment，需要將這兩個segments做union的動作。
