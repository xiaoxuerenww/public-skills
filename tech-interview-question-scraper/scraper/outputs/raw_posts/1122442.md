## Post

年初LinkedIn收到hm私信，这个组是之前数据砖收购的某ml platform公司人马。
hm 初筛 - 问了过去项目经历。hm提到这个org面试有两种选择，ml track 和 traditional backend track，面试内容不一样，hm表示选哪个track 都行，主要是希望找good engineer。我面的是traditional backend
以下内容需要积分高于 200 您已经可以浏览
电面: 去office通勤题，follow up是可以换交通方式
VO system programming: design snapshot kv store, 需要支持 add/delete/get/get_at_a_snapshot，需要run test cases并且自己再加一些test cases
VO coding: primary/fallback circuit breaker，需要自己写一些test cases
VO SD: 设计一个 compute platform，假设我们管理着X个GPU，用户可以提交不同优先级的job到我们平台来运行。感觉这轮面试官希望我说一些Kubernetes的东西，但是我完全没经验也不好瞎说。所以这就说了一些job queue， docker repository， fault tolerance之类的。
VO hm - 这轮体验超级差，还是之前那个hm，全程没耐心，很condescending的态度。因为我背景和这个组不算特别match，这个初筛的时候我就和hm说了，他表示完全可以来了再学。但是这轮面试时候态度大转弯，全程非常不耐烦。
面完快两周也没消息，肯定是挂了，也懒得问了
面下来感觉 engineer都很厉害也很nice，虽然很忙但是面试的时候都很认真交流认真听
recruiter非常拉垮，基本不回任何消息。hm态度傲慢且不耐烦。
本来是对这个面试期待很高，花了很久认真准备了地里的面经，这个结果只能说互相放过吧。
最后，求米看面经！！
