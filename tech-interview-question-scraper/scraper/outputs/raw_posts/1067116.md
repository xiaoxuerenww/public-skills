因为数的结构和定义是固定的，给出order，容易算出任何子树的size。通过判断size和目标直接大小，递归算出路径。复杂度是线性。
cache = {}
def fib_tree_size(order):
if order in cache:
return cache[order]
if order <= 1:
cache[order] = 1
else:
cache[order] = fib_tree_size(order - 2) + fib_tree_size(order - 1) + 1
return cache[order]
def fib_path(order, source, dest):
if source == dest:
return ''
if source < 0 or source >= fib_tree_size(order):
raise Exception("source out of bound")
if dest < 0 or dest >= fib_tree_size(order):
raise Exception("dest out of bound")
if order <= 1:
raise Exception("error")
skip_left = fib_tree_size(order - 2) + 1
if dest == 0:
path = fib_path(order, dest, source)
return 'U' * len(path)
elif source == 0:
if on_left_subtree(order, dest):
return 'L' + fib_path(order - 2, source, dest - 1) # this needs be optimized, otherwise its O(n^2)
else:
return 'R' + fib_path(order - 1, source, dest - skip_left)
else:
s_left, d_left = on_left_subtree(order, source), on_left_subtree(order, dest)
if s_left and d_left:
return fib_path(order - 2, source - 1, dest - 1)
elif not s_left and not d_left:
return fib_path(order - 1, source - skip_left, dest - skip_left)
else:
return fib_path(order, source, 0) + fib_path(order, 0, dest)
def on_left_subtree(order, dest):
return dest >= 1 and dest <= fib_tree_size(order - 2)
print(fib_path(5, 5, 7))

I got the same question. I was able to complete the question with a lot of help from the interviewer. It is possible to do the question without building the tree. Basically, the question is similar to finding the path between two nodes in a tree but in this case you don't have the tree itself. Imagine if you had the tree, how would you do it? You would probably go from root to root.left or root.right right? Now, Since we don't actually have the tree, we know the root is 0. Now how to determine what root.left is and what root.right is? root.left is essentially root+1 i.e. 0's left is 1. And what is root.right? It is number of nodes in left subtree of root+1. Number of nodes in left subtree is fib(n-2) where n is order. You will see if you just write a normal function assuming there is a tree and then replace root, root.left and root.right with numbers.
