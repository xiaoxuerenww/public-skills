## Post

刚刚做完。想给大家分享点经验和心得。
题目大概就是个 in memory 的 DB 管理系统，支持 CRUD。但其实用 Hashmap 就行。一共90分钟，4个问题。每一个都是 based 在前一个问题上的。给我的邮件里明确写着，必须要用
Python
。其他语言提交的会被直接无视。
逻辑很简单，不需要用到啥算法。跟Leetcode一比，逻辑思维的难度应该算偏简单的。
但整体仍然不容易。因为首先题就巨长，特别长。。。就跟读小作文似的。还有就是时间特别紧迫。我只做到了第三题，还只是过了大约 3/4 的 Test Case。第四题就没碰到。所以默认应该是挂了。
假如我能重来一遍，我会注意一下几点：
注意 Return Type！有些很简单的逻辑，我想当然觉得 inline 更改完就好了，等等。但其实人家让你 return 一些结果的。不要忘了。
完全不用
在意 code 的 readablity —— 能写出来就行，简单粗暴的过 Test Cases 就可以了。没人会看你的代码的。
快。一定要快。。。。读题要快，然后写码的手速要快。。
审题。就有些 Edge Case，或者条件，题目里要求的。没啥逻辑，但是如果没读到或者忽略了，就不满足要求了。
其他一些 Tips
我仔细查了一下，CodeSignal 应该是后台会自动监测复制粘贴的 Activity 的。所以哪怕没有 Proctor ，如果直接复制 prompt 然后让 ChatGPT 解决再复制代码 —— 这个操作有可能就被判作弊了 【但是我不 100% 确定。求证】
在做题之前，可以简单花几分钟 Practice 一下，熟悉 Code Signal 的 界面。
求加米。
