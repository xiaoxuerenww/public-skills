## Post

是很常见的 SnapshotSet Iterator
一开始听到这题大喜 刷面经刷得很熟了
于是我把两种方法都讲了遍 一种是 global log. 一个array of log 写到底
另一种是per key versioned log. 维持一个dictionary，value 是 version log
并且两种优缺点都讲了
per key versioned log 好处是可以 bisect log. 但是initialize iterator 时需要复制key. 不能直接指向原本的 per key version log. 因为指针会乱掉
global log 好处是iterator initialize 时 不用复制任何东西 Time complexity 是 O(1)
讲完之后面试官好像不太满意 我先题了per key versioned log 面试官说能不能优化initialization 于是我提了global log 讲完面试官又说那假设log 很长怎么办，我就说那可能只能用 per key versioned log.  他让我详细讲解version set  怎么freeze 还有怎么iterate. 自认为都讲的非常清楚，但面试官不知道哪里不满意 直到最后10 分钟他说时间不多了 就照你说的写吧 才匆匆写了 per key version log的版本 最后bisect 部分没写完
最后果然挂了
想问问大佬们这题还有更优的解法吗？ 真心不懂挂在哪
