## Post

电面了两轮
第1轮可能是面试官的新题 非常难了
给定一组 node groups（如 [[1],[2,3],[4,5,6]]），将所有 group 连成一个连通图。连接方式：从两个不同 group 各随机选一个节点，建一条边。要求最终整个图连通，用最少的边（k 个 group 需要 k-1 条边）要求随机
暴力就是去枚举 次优即便最小生成树算法Kruskal 也不是 uniform random spanning tree——某些树结构的出现概率更高 需要更高级的算法（不常规）
Input: groups = [[1], [2, 3], [4, 5, 6]]
Output: [(1, 3), (2, 5)]  (随机结果之一)
or [(1, 6), (3, 4)]  (另一种可能)
第二轮是经典给一个 2D grid（S=起点，D=终点，X=障碍，1-4 代表 Walk/Bike/Car/Train），加上 cost[] 和 time[] 矩阵。只能水平/垂直移动，不能对角。基础版限定同一 mode 走到底，找最快的 mode（tie-break 最便宜）。Follow-up 允许切换 mode（有额外 penalty），再限制切换次数。
要么4次BFS 要么BFS把1-4也加入队列 两种都行 面试官可能希望第二种
followup 迪杰斯特拉没让我写 太复杂了存了都好几个state了 大致说了说思路
最后过了但是没HC 一直hold没法进行下一步 说有了再联系我 然鹅已经时过境迁。。。
