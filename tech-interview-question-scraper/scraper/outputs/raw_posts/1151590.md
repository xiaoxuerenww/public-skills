## Post

内推投递，fresh new grad 1+1+4轮新鲜真实挂经，请各位看官老爷们走过路过加加米，小弟没大米，感激不尽！🙏
第一轮 hr call: 一般一个学校应该是同一个人，应该是个高级别的recruiter，问why dbx，career goals，timeline，offer ddl，之前有别的公司实习会问有没有return，然后最牛的project，自我介绍等常规内容，一般不挂人，但也听说有人挂的
第二轮 prescreen：
地里常见的CIDR题，给一个list的rules和ip，格式类似[{"allow", CIDR_1}, {"deny", CIDR_2}]，找第一个匹配的（即在范围内的），看是allow还是deny；followup是把ip换成cidr，即代表一段范围，全部被allow的范围覆盖才allow，否则deny
back2back四轮
coding：面试官一开始说不强调时间复杂度，关注代码质量与可维护性，适当考虑内存，且不要过度engineering，事实证明必须一次性写出时空最优的才行，说的跟白说一样，不能用trivial solution lol。。
题是地里有的，但不是很常见，实现一个压缩一堆正整数的encoder，把 32 位整数序列压缩成一组“run”，每个 run 使用两种编码方案之一，可以交错使用，类似[RLE, BP, RLE...]。给的输入的原始顺序不可改变。而且必须流式处理，也就是不能储存原输入。RLE：把连续相同的值编码为
(value, count)
复制代码
。要求必须大于等于8个值一组。（但是最后的几个数可以少于8个数）BP：把恰好 8 个值打包为一段，值可以不同。BP 适用于没有满足 RLE 最小计数要求的地方。（也是最后几个数可以少于8个数），难点在于流式处理，然后最后几个数怎么处理（你不知道接下来会有多少数，所以任何目前最新的输入都是最后的数）。给一大堆struct和class，严格用给的类来实现。时间很紧。。一开始面试官还非要聊天，然后讲题目讲了二十分钟。。最后卡点写完，只测了最基本的没啥问题，但可能有edge case的bug，不知道是不是挂在这轮了，有点无语
cross-function：常规问题，conflict，ambiguous situation，ownership，feedback等，30分钟草草结束
system design：叫设计一个类似waymo的打无人车的与用户交互的app，关注用户交互，问的很细，经常打断问design原因，细到需要把每个请求的payload写出来，request和response，get or post这种，没要画流程图，但是需要说出用哪些表，schema，需要哪些service，还得考虑到系统万一崩了，请求峰值，原子性操作，用rest还是rpc还是websocket，打车的时候给的价格怎么lock price，异步同步问题，一个service需要哪些api，怎么处理用户cancel，调度车等问题。。答的7788吧
algo: 最舒服的一轮，面试官也很友善，20分钟基本就做完且最优了，非题库题，大概是给两个string，第一个是需要被match的字符串，第二个是query，用来match第一个的substr的，找出match到的substr的第一个字符的下标，substr的内部顺序可以不同，类似[databricks, tada] -> 0; [databricks, ribkc] -> 4; [databricks, abc] -> -1; [data, databricks] -> -1; 同样需要考虑空字符串等edge cases
walk-out：跟第一轮的hr一样，问问面试体验等
第二天下午就挂了。。也没给啥feedback，大概意思就是多面面就有经验了hh（菜就多练）
总体体验是比大厂难，而且都是聪明人+卷怪，bar还是挺高的，就当攒经验了
有问题欢迎讨论，感谢大家加米！！
