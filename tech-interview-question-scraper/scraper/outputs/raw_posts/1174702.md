## Post

店面是q1 image processing，用 PIL 做 grayscale、scale、resize，第一步处理小图，第二步处理大图，improve performance 用的 ProcessPoolExecutor。recruiter 事先提了一句要熟悉 parallelism/concurrency，算是提示了。时间有点紧，面试官还提醒了一下才做完。整体还行，过了。
昂赛几轮，记录一下。
SD 那轮有点意思，让设计 1 对 1 的聊天系统，面试官强调只需要 1-1，group chat、channel 都不在范围内。地里之前好像没见过这道题，不知道是新题还是面试官心血来潮。备考时参考了这个 chat system 解法，原题是 Slack 的设计，带 group 和 channel，1-1 把那部分去掉就好，核心思路还是相通的。聊了收件方不在线怎么推消息、怎么知道用户上线、session 怎么保存这些，然后问到 Kafka vs Redis，Kafka 的原理是什么，每部分问得挺细。复盘了一下，Kafka 那块底层没搞透，被深挖的时候很被动。
Coding 是 file dedup，通过 file size 和 content hash 去重。本题不难，但 follow-up 很深，问了 IO bound 还是 CPU bound 怎么区分、大文件怎么处理、大量文件的场景、realtime 检测需要什么机制。我有几个 follow-up 没答利索，题本身不难，结果反而死在 follow-up 上了。
Culture 那轮比 technical 难猜，面试官问为什么来 Anthropic，要说到具体的 value 和自己经历过的例子，不能泛泛讲对 AI 感兴趣。还问了 AI safety 的看法、遇到过的 moral dilemma。全程很难判断他在打几分，猜不透他想要什么样的回答。
HM 比较轻松，主要聊了复杂项目经历和 roadmap 影响力，还问了怎么 mentor 别人。面试官有点 cold，聊了四十多分钟感觉全程都在给对面单口相声，不太知道往哪个方向走。
等结果，大概率 culture 那轮是我的软肋，希望能过吧。求米。
